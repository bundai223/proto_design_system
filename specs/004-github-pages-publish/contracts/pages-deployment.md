# Contract: GitHub Pages Deployment Surface

## Purpose

Define the repository-level contract for building and publishing the design system showcase to GitHub Pages.

## Inputs

### Source Surface

- Repository source for the visitor-facing site comes from `demo/`.
- Vite configuration is sourced from [`vite.config.ts`](/home/bundai223/repos/github.com/bundai223/proto_design_system/vite.config.ts).
- Package scripts are sourced from [`package.json`](/home/bundai223/repos/github.com/bundai223/proto_design_system/package.json).

### Build Invocation

- Local build command: `npm run demo:build`
- CI build command: `npm run demo:build`
- Optional path override: `PAGES_BASE_PATH=/custom-path/ npm run demo:build`

### Hosting Context

- Expected project Pages URL form: `https://bundai223.github.io/proto_design_system/`
- Expected project Pages base path: `/proto_design_system/`
- Base path must default to `/proto_design_system/` and remain overrideable at build time.

## Outputs

### Publish Directory

- Build output directory: `dist/demo`
- Required entry file: `dist/demo/index.html`
- Required asset directory: `dist/demo/assets/`

### Deployment Behavior

- The workflow must upload only the Pages publish directory, not the full repository or unrelated generated files.
- The deployment must publish through GitHub Pages Actions rather than by committing generated output to a long-lived deploy branch.
- Workflow failure must be visible in GitHub Actions status.
- The workflow must run on pushes to the default branch and support manual dispatch.

## Validation Rules

- `dist/demo/index.html` must reference built assets using the configured Pages base path rather than domain-root-relative URLs when in project Pages mode.
- The build output used locally and in CI must be generated by the same command.
- A successful workflow run must result in a Pages deployment associated with the repository.
- Changing the build-time base path must not require edits under `demo/`.

## Out of Scope

- Custom domain setup
- Multi-environment deployments
- Separate marketing or documentation site outside the current `demo/` showcase
